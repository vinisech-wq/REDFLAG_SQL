# REDFLAG_SQL
SQL-based Fraud Detection System that analyzes transaction data to identify suspicious activities such as velocity fraud, card testing, refund abuse, mule accounts, merchant collusion, structuring, account takeover, and geographic impossibility. Built using MySQL with SQL concepts including joins, subqueries, CTEs, window functions.

# RedFlag: SQL Fraud Detection Engine

## Overview

RedFlag is a SQL-based fraud detection project developed as part of The Unlox Academy Minor Project. The objective is to identify suspicious transaction patterns from a dataset containing approximately 200,000 financial transactions generated by a fictional Indian payment aggregator, PayFast. The project demonstrates how SQL can be used to solve real-world fraud detection problems without relying on machine learning or external programming languages. RedFlag_Project_Brief.pdf

## Problem Statement

Financial fraud is often hidden within large volumes of transaction data. The goal of this project is to analyze transactional behaviour and identify users or merchants exhibiting suspicious patterns using only SQL queries.

## Dataset

Database: MySQL

Table: `transactions`

Records: Approximately 200,000

Time Period: January 2024 to June 2024

Users: Approximately 14,700

Merchants: 800

The dataset contains transaction details such as transaction ID, user ID, merchant ID, amount, transaction time, payment mode, transaction status, city, and transaction type. redflag_transactions.sql

## Fraud Patterns Detected

1. Velocity Fraud
2. Round Amount Clustering
3. Card Testing
4. Failed Transaction Detection
5. Odd Hour Concentration
6. Mule Accounts
7. Refund Abuse
8. Merchant Collusion
9. Just Under Threshold (Structuring)
10. Dormant Then Active Accounts
11. Velocity Spike
12. Geographic Impossibility
